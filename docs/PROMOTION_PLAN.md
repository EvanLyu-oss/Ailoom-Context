# Promotion Plan

This plan is for growing Ailoom Context from early GitHub stars into a credible open-source beta with real user feedback.

Current baseline:

- GitHub stars: 7
- Product stage: v1 beta
- Primary ask: try the tool on a real project and send feedback
- Repository: `https://github.com/EvanLyu-oss/Ailoom-Context`

## Positioning

Primary positioning:

```text
Local-first context compression for AI coding agents.
```

Supporting points:

- Compact AI-readable skeletons.
- Exact local restore.
- No cloud upload or telemetry.
- Token savings and speed diagnostics.
- Daily handoff reuse for unchanged projects.
- Patch/replay and benchmark harness for serious workflows.

## Target Users

Best-fit early users:

- Developers using AI coding agents on large repositories.
- Users of Cursor, Codex, Claude Code, Cline, Continue, OpenHands, and similar tools.
- Developers who care about local-first workflows and source privacy.
- Writers or researchers working with long manuscripts.

Avoid spending early effort on:

- Users expecting a hosted SaaS product.
- Tiny-project users who may not see token savings.
- Enterprise procurement before v1.0 stability messaging is ready.

## 30-Day Goals

Conservative goals:

- 50 GitHub stars.
- 10 real trial reports or detailed feedback messages.
- 3 public user quotes.
- 2 reproducible third-party trial examples.
- 1 short demo video or animated screenshot.

Stretch goals:

- 150 GitHub stars.
- 30 trial users.
- One community post with meaningful discussion.
- One integration request from an AI/IDE workflow user.

## Week 1: Prepare The Surface

Owner: Codex

- Keep README first screen clear and outcome-focused.
- Publish `GOOD_FIRST_TRIAL.md`.
- Publish `docs/SHOWCASE.md`.
- Publish `docs/LAUNCH_POSTS.md`.
- Publish this promotion plan.
- Make sure feedback email and repository URL are visible.

Owner: Evan

- Prepare 2-3 screenshots of successful `ailoom handoff`, `ailoom next`, and `ailoom savings`.
- Collect one or two honest quotes from friends who tested it.
- Decide which personal channels to post first.

## Week 2: Soft Launch

Goal: get feedback without overexposing the project.

Channels:

- Personal WeChat / friend circle.
- Developer friends and small groups.
- GitHub repository topics.
- One Chinese developer community post.
- One English developer community post.

Message:

```text
I am looking for beta feedback from people using AI coding agents on real projects.
```

Do not overclaim. Ask for trials.

## Week 3: Public Launch

Only do this after at least a few real trial reports.

Channels:

- Hacker News Show HN.
- Reddit developer communities where self-promotion is allowed.
- Dev.to or similar technical article.
- V2EX / 掘金 / 知乎.
- X/Twitter or LinkedIn thread.

Content angle:

- "Local-first context compression for AI coding agents."
- "Why context compression should keep exact restore local."
- "Dogfooding Ailoom Context on itself."

## Week 4: Convert Feedback Into Roadmap

Track:

- Install failures.
- Confusing first command.
- Cases where token savings are weak.
- Cases where agent understanding improves.
- Requests for IDE integration.
- Requests for package-manager install.

Turn feedback into:

- GitHub issues.
- README FAQ improvements.
- v1.0 blockers.
- Integration roadmap.

## Metrics To Track

Weekly:

- GitHub stars.
- GitHub forks.
- Issues opened.
- Trial reports received.
- Install success/failure count.
- Number of users who ran a real project handoff.
- Number of users who sent `ailoom savings` output.
- Repeated-user feedback: "Would you keep this in your workflow?"

## Messaging Rules

Use:

- "observed"
- "current beta"
- "early testers reported"
- "local-first"
- "exact restore"

Avoid:

- "guaranteed 10x"
- "always saves 94%"
- "replaces your AI IDE"
- "secure by magic"
